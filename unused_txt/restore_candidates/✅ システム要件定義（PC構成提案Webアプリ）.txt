✅ システム要件定義（PC構成提案Webアプリ）

版数: v1.1
最終更新日: 2026-03-14

1. 目的
ユーザーが予算と用途（ゲーミングPC、動画編集PC、汎用PC）を入力すると、
互換性を考慮した最適なPC構成を自動提案し、購入リンクを表示する。

2. スコープ
対象（MVP）
- 予算と用途から構成を1案生成する
- 主要パーツ（CPU/GPU/メモリ/ストレージ/電源/ケース）を返す
- 互換性のある構成のみ提示する
- 購入リンク（主データソース）を表示する

対象外（MVPでは実施しない）
- ユーザー認証/マイページ
- ベンチマーク指標の詳細可視化
- 複数候補のランキング表示

3. 機能要件
3.1 必須機能

予算入力機能
- ユーザーが日本円で予算を入力できる。
- 入力範囲は 50,000 円以上 500,000 円以下とする。

用途選択機能
- ゲーミングPC / 動画編集PC / 汎用PC を選択できる。
- API内部の列挙値は gaming / video_editing / general とする。

構成生成機能
- 予算配分アルゴリズムでカテゴリ別に予算を配分する。
- 互換性チェックを行う（CPUソケット、メモリ規格、電源容量、ケースサイズ）。
- 互換性を満たさない場合は候補を再探索する。

パーツ情報取得機能
- ドスパラ PCパーツページ（https://www.dospara.co.jp/parts）を主データソースとしてスクレイピングする。
- 比較サイトへのリンクを表示できる（任意）。
- Amazon/Rakuten のリンクを表示できる（任意）。

結果表示機能
- 構成一覧（CPU、GPU、メモリ、ストレージ、電源、ケース）を表示する。
- 価格合計を表示する。
- 購入リンクを表示する。
- 最終更新日時（データ鮮度）を表示する。

3.2 追加機能（将来的に）
- ユーザー登録とお気に入り構成保存
- 性能指標（PassMark、ベンチマーク）表示
- 在庫状況チェック

4. 互換性チェック要件
4.1 必須条件
- CPU とマザーボードのソケットが一致すること。
- メモリ規格（DDR4/DDR5）がマザーボード対応規格と一致すること。
- 電源容量が推奨消費電力合計の 1.2 倍以上であること。
- ケース対応フォームファクタ（ATX/mATX/ITX）にマザーボードが適合すること。
- GPU長がケースの最大GPU長以内であること。

4.2 推奨条件
- メモリ容量は用途別最低基準を満たすこと（gaming: 16GB, video_editing: 32GB, general: 16GB）。
- ストレージはNVMe SSDを優先すること。

4.3 不整合時の扱い
- 不整合時は別候補へ置換し、再評価する。
- 最終的に成立しない場合はエラーコード compat_no_solution を返す。

5. 非機能要件
- パフォーマンス: 構成生成 API のP95応答時間は 3秒以内。
- スケーラビリティ: 同時アクセス100ユーザーで主要機能が利用可能。
- セキュリティ: HTTPS通信、API認証（JWT）、入力バリデーション、レート制限。
- 可用性: 月間稼働率99%以上。
- 可観測性: APIエラー率、応答時間、スクレイピング成功率を監視する。

6. 技術要件（Django統一）
- フロントエンド: React
- バックエンド: Django + Django REST Framework
- DB: PostgreSQL（パーツ情報キャッシュ）
- スクレイピング: Python（requests + BeautifulSoup）
- 定期実行: Django管理コマンド + Cron（将来はCelery Beatへ拡張可）
- 構成生成アルゴリズム: Python（Djangoアプリ内サービス層で実装）
- ホスティング: AWS（EC2 + RDS）

7. インターフェース要件
7.1 UIフロー
- ホーム画面 -> 予算入力フォーム -> 用途選択 -> 「構成を生成」ボタン
- 結果画面 -> パーツ一覧 + 購入リンク表示
- 異常系 -> エラーメッセージ + 再試行導線

7.2 API設計

POST /api/generate-config
- リクエスト
	{
		"budget": 100000,
		"usage": "gaming"
	}
- バリデーション
	- budget: 50000 以上 500000 以下の整数
	- usage: gaming | video_editing | general
- 成功レスポンス（200）
	{
		"config_id": "cfg_20260314_0001",
		"usage": "gaming",
		"budget": 100000,
		"total_price": 98760,
		"generated_at": "2026-03-14T10:20:30+09:00",
		"parts": [
			{
				"category": "cpu",
				"name": "sample cpu",
				"price": 24800,
				"shop": "dospara",
				"url": "https://example.com/item/cpu"
			}
		],
		"warnings": []
	}
- 失敗レスポンス（400）
	{
		"error": {
			"code": "invalid_input",
			"message": "budget must be between 50000 and 500000",
			"details": {
				"field": "budget"
			}
		}
	}
- 失敗レスポンス（422）
	{
		"error": {
			"code": "compat_no_solution",
			"message": "No compatible configuration found under given constraints"
		}
	}

GET /api/parts
- クエリ
	- category: cpu | gpu | memory | storage | psu | case
	- page: 1以上の整数
	- page_size: 1-100
- 成功レスポンス（200）
	{
		"count": 123,
		"page": 1,
		"page_size": 20,
		"results": [
			{
				"id": 1,
				"category": "cpu",
				"name": "sample cpu",
				"price": 24800,
				"updated_at": "2026-03-14T06:00:00+09:00"
			}
		]
	}

8. データ更新・運用要件
- パーツ情報更新は1日1回、毎日06:00(JST)に実行する。
- スクレイピング失敗時は3回まで指数バックオフで再試行する。
- 全再試行失敗時は前回成功データを提供し、運用アラートを通知する。
- APIレスポンスにはデータ最終更新日時を含める。

9. 制約条件
- ドスパラの利用規約に従う（スクレイピング頻度制御）
- robots.txt と利用条件を確認した上で収集を行う
- 日本円対応
- 取得不可時は代替ソースまたは前回キャッシュで継続提供する

10. 受け入れ基準（Acceptance Criteria）
- AC-01: budget=100000, usage=gaming の入力に対して、6カテゴリを含む構成を返す。
- AC-02: 互換性不整合の構成は結果に含まれない。
- AC-03: 正常系で total_price は budget 以下になる。
- AC-04: 構成生成APIのP95応答時間が3秒以内。
- AC-05: 不正入力時にエラーコード invalid_input を返す。
- AC-06: 解なしの場合にエラーコード compat_no_solution を返す。

11. 開発ステップ
1) 構成生成アルゴリズム試作（仮データ）
2) スクレイピング実装（CPU/GPU -> DB保存）
3) Django REST FrameworkでAPI構築
4) UI開発（React）
5) 統合テストとデプロイ
