InterfaceTypeテーブル要件

識別情報
id: BIGSERIAL, 主キー
name: VARCHAR(30), NOT NULL, UNIQUE

説明
マスターデータ（I/F タイプ参照テーブル）
ストレージやその他パーツのインターフェース規格を標準化。
例: SATA, NVMe/PCIe 3.0, PCIe 4.0, USB 3.0 など

用途
- Storage: SSD/HDD の接続規格（SATA, NVMe など）
- GPU: GPU バス規格（PCIe x16 など）
- その他周辺機器

制約
name は NULL 不可、UNIQUE
I/F タイプ名の一意性

推奨インデックス
INDEX name（UNIQUE に含まれる）

PostgreSQL DDLサンプル

CREATE TABLE interface_type (
  id BIGSERIAL PRIMARY KEY,
  name VARCHAR(30) NOT NULL UNIQUE
);
